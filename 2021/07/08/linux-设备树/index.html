linux 设备树

Qemu Device Tree

1. 设备树的概念

设备树是描述计算机的特定硬件设备信息的数据结构，以便操作系统内核可以方便的管理和使用这些硬件，包括CPU、内存、PCIe总线和其他一些外设。嵌入式设备的引导过程中，通常使用设备树来传递无法被自动探测的板级硬件信息。

2. 设备树的文件

2.1 文件之间的关系

设备树文件的源文件是后缀为.dts（Device Tree Source）文件，每一款单独的硬件可以单独写一份。.dts文件编译后得到的可执行文件是.dtb（Device Tree Blob）文件，dtb文件通过Bootloader引导程序加载到内核。

2.3 dts文件的基本框架

设备树由节点组成，其中的每一个节点都有如下几个部分：

  1. 根结点：“/”

  2. 节点名称：如“node1”和“node2”

  3. 节点地址：

  4. 子节点：如“child-node1”

  5. 属性：名称+属性值

    属性实际上是一对对的键值对，属性名为键，属性值为值。其中，值有五种可能的方式：

    1. 文字字符串（text strings）
      • string-property = "a string";
    2. Cells（32 bit unsigned integers delimited by a angle brackets）
      • cell-property = <0xbeef 123 0xabcd1234>;
    3. Binary data is delimited with square brackets
      • [0x01 0x23 0x45 0x67];
    4. 不同的表示方式可以混合，用逗号隔开：
      • mixed-property = "a string", [0x10 0x23 0x45 0x67], <0x1234>;
    5. 逗号还可以用来创造一组字符串：
      • string-list = "red fish", "blue fish";

  6. 标签

2.4 基本概念

2.4.1 compatible

comatible是每一个设备的基本属性，他表示了设备的标识符，每一个设备应该拥有一个独一无二的compatible。

  • compatible的格式：“<manufacturer>,<model>”

⚠️值得注意的是，compatible不一定只有一个参数，有可能写作一个字符串列表，例如compatible = "fsl, mpc8349-uart", "ns16550"。这时候，第一个字符串表示的是该设备的准确类型（exact device），之后的表示该设备可兼容的其他设备。

2.4.2 节点名称

  • 节点名称的格式：<name>[@<unit-address>]

  • name：ascii 字符串且不超过31个字符，一般情况下用来表示设备的种类，例如cpu、ethernet

  • unit-address：用来获取该设备的原始地址，他会被列入节点的reg属性中

2.4.3 寻址

  • 寻址使用的三个属性：

    1. reg
    2. #address-cells
    3. #size-cells

  • reg

    • reg的基本格式：reg = <address1 length1 [address2 length2] [address3 length3] ... >

  • #address-cells 和 #size-cells 则用来表示子模块的寻址的reg的属性

  • ```dts
    / {

    #address-cells = <1>;
#size-cells = <1>;

...

serial@101f0000 {
    compatible = "arm,pl011";
    reg = <0x101f0000 0x1000 >;
};

    };

  • 上述另一个例子中，地址由两个数字表示，那么第一个数字表示chipselect number，第二个数字表示offset。

Non Memory Mapped Devices

非内存映射的设备使用时，#size-cells属性为0，仅根据具体的地址号寻址。

2.4.4 地址转换（ranges）

设备树中使用ranges表示地址的转换。例如，从CPU地址转换到PCI总线地址的时候，我们需要将CPU的地址转换成具体的PCI地址，从而达到访问设备的目的。在下面这个例子中，external-bus拥有独立的地址空间，设备ethernet的总线地址为0，长度为0x1000，会被总线转换成0x10100000-0x1010ffff。

2.4.5 中断

四个属性与中断相关：

  1. interrupt-controller：空属性，表示一个节点是中断控制器
  2. #interrupt-cells：中断控制器的属性，表示多少个数字作为中断标识符（与#address-cells和#size-cells类似）
  3. interrupt-parent：表示一个节点指向的中断控制器
  4. interrupts：中断标识符，一个标识符代表一个信号

2.4.6 特殊节点

  1. aliases：表示别名

  2. chosen：非实际设备，表示一个从firmware到os传递数据的空间。

2.4.7 PCI

  1. PCI Bus numbering

    bus-range：包含两个数字，第一个表示赋给该节点的bus number，第二个表示任何下属的PCI总线的最大总线编号。

    • phys.mid cell: hhhhhhhh hhhhhhhh hhhhhhhh hhhhhhhh

    • t：aliased address flag

      • 01：I/O space
      • 10：32 bit memory space

    • ddddd：device number
    • fff：function number, used for multifunction PCI devices
    • rrrrrrrr：Register number, used for aconfiguration cycles

3. Qemu中的device tree

3.1 api

dtb一词在qemu中被fdt（Flat Device Tree）一词取代，以下所有提到fdt的地方指代dtb，linux平台为设备树的创建提供了libfdt库，由libfdt-devel rpm包提供，这个库提供了fdt操作的各种接口，部分如下：

  • fdt_create：初始化设备树
  • fdt_add_subnode：设备树中添加节点
  • fdt_setprop_cell：设置设备树属性键值对，值为数字
  • fdt_setprop_string：设置设备属性键值对，值为字符串

3.2 wrapper api

qemu对libfdt的接口做了二次封装，部分接口举例如下：

  • create_device_tree：初始化一颗设备树
  • qemu_fdt_setprop_cell：设置设备树的节点的属性，值为数字
  • qemu_fdt_setprop_string：设置设备树的节点的属性，值为字符串
  • qemu_fdt_add_subnode：添加一个子节点

3.3 设备节点创建及加载

根据此前的设备树的基本规则，我们在qemu中可以创建一系列的设备树节点，最后形成一个完整的设备树。形成的方式各个开发板的形式各不相同，Arm架构将其保存在一个结构体中，而ppce500则没有。

qemu在加载好全部镜像之后，跳转到kernel之前，会先加载一个临时生成的bootloader固件到内存，然后跳转到这个固件的执行代码，最终通过这个固件跳转到内核。这就是此前所说的加载dtb文件的过程。
